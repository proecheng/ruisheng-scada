## 3. Q-D03 — 波形 BLOB 编码格式

### 结论
`DataArry` 字段是 **大端序二进制流**，固定 2 字节头 + 变长数据体：

```
偏移  字段       长度  说明
0     SampleTime  1B   采样间隔（单位 0.1 秒；值 10 = 1 秒）
1     PacketCnt   1B   本 BLOB 内的采样点数
2..   Values[N]   变长 按 DevPointInf.ValueType:
                      "字"   → 2B 大端无符号整数
                      "双字" → 4B 大端无符号整数
                      N = PacketCnt
```

时间戳反推：第 i 个采样点的时间 = `EnterTime - (PacketCnt - i) × SampleTime × 0.1 秒`

### 证据

**文件**：`D:\江苏润盛\济南大学开发软件\WEB\ModBusWeb20210811\IOTWeb\Ajax\GetChartData.aspx.cs:703-733`

关键代码片段：

```csharp
int sampleTime = bytes[0];
int packetCnt  = bytes[1];
int pos = 2;
for (int i = 0; i < packetCnt; i++) {
    if (type == "字")   value = (bytes[pos++] << 8) | bytes[pos++];
    if (type == "双字") value = (bytes[pos++] << 24) | (bytes[pos++] << 16)
                              | (bytes[pos++] << 8)  | bytes[pos++];
    ts = enterTime.AddSeconds(-(packetCnt - i) * sampleTime * 0.1);
}
```

### 影响
- §4.2.C 补充了完整字节布局和反序列化伪代码（已回填）

### 遗留
- **字节序确认**：代码里是大端（高字节在前），这与 ModBus 协议习惯一致，但与 x86 机器字节序相反 —— 迁移新代码时要注意小心用 `IPAddress.NetworkToHostOrder` 或类似的转换
- 浮点数怎么表示？代码里只看到整数读取，**如果有浮点点位，可能用定点数（原值 × Ratio）表示**，建议抓一个真实 BLOB 核对

## 4. Q-B01 — PhoneAlarm 位定义

### 结论
`PhoneAlarm` 是 16 位整数，分四段位域：

| 位段 | 公式 | 含义 |
|---|---|---|
| bit 0 (0x0001) | `PhoneAlarm & 0x01` | 触发告警时是否打电话 |
| bit 1 (0x0002) | `PhoneAlarm & 0x02` | 恢复/复位时是否打电话 |
| bit 2–7 | — | 保留（未使用）|
| bit 8–11 | `(PhoneAlarm >> 8) & 0xF` | 触发时动作码 |
| bit 12–15 | `(PhoneAlarm >> 12) & 0xF` | 恢复时动作码 |

**动作码枚举**：

| 值 | 语义 |
|---|---|
| 0 | 无动作 |
| 1 | 全开（所有继电器 ON） |
| 2 | 全关（所有继电器 OFF） |
| 3 | 本路开（告警点位对应继电器 ON） |
| 4 | 本路关（告警点位对应继电器 OFF） |

### 证据

**文件**：`D:\江苏润盛\济南大学开发软件\WEB\ModBusWeb20210811\IOTWeb\PhoneCfgLimit.aspx`

关键行号：
- **第 142–198 行**：前端下拉框逐位渲染 PhoneAlarm 当前值
- **第 167 行**：`(PhoneAlarm >> 8) & 0xF` 解析触发动作码
- **第 198 行**：`(PhoneAlarm >> 12) & 0xF` 解析恢复动作码
- **第 282–299 行**：表单提交时按位重新拼装 PhoneAlarm

### 影响
- §F.4 重写为完整位定义表（已回填）

### 遗留
- 其他渠道（短信/微信/邮件）**不走 PhoneAlarm 位**，而是由 `UserAlarmInf.PhoneList / EmailList / WXList` 有无决定 —— 这点已补入 §F.4 注释
- `ResetRemind=1` 时复位通知走哪些渠道，仍是 **Q-B04** 待澄清

## 5. Q-B05 — ControlAuthority 位掩码

### 结论
`ControlAuthority` 是**位掩码**（不是单值枚举）：

| 位 | 权限 | 判断 |
|---|---|---|
| bit 0 (0x01) | 设备控制权（远程启停、下发命令） | `(CA & 0x01) != 0` |
| bit 1 (0x02) | 配置管理权（改阈值、改通道名、加删设备） | `(CA & 0x02) != 0` |
| bit 2–7 | 预留 | — |

**典型取值**：`0` = 只读、`1` = 可控制、`2` = 可配置、`3` = 全权

### 证据

if ((LogUser.ControlAuthority & 0x02) == 0x00) { /* 无管理权 */ }

## 6. Q-P07 — TCP 是否用标准 MBAP？（部分澄清）

### 结论（部分）
- **TCP 6020 上**：代码中有标准 ModBus CRC16（多项式 `0xA001`），**没有** MBAP Header 处理代码 → 强烈倾向 **TCP 上直跑 RTU 帧（含 CRC16）**
- **TCP 6030 上**：纯 UTF8 字符串（JSON），**不是** ModBus TCP

### 证据

**CRC16 实现**：`D:\江苏润盛\济南大学开发软件\设备录入工具\电能终端波形分析工具\Form1.cs:116-142`

```csharp
private ushort CRC16_Check(byte[] data, int length) {
    ushort Reg_CRC = 0xFFFF;
    for (int i = 0; i < length; i++) {
        Reg_CRC ^= data[i];
        for (int j = 0; j < 8; j++) {
            if ((Reg_CRC & 0x0001) == 0x0001)
                Reg_CRC = (ushort)(Reg_CRC >> 1 ^ 0xA001);  // ModBus RTU 多项式
            else
                Reg_CRC = (ushort)(Reg_CRC >> 1);
        }
    }
    return Reg_CRC;
}
```

**TCP 6030 字符串传输**：`D:\江苏润盛\济南大学开发软件\WEB\ModBusWeb20210811\IOTWeb\DataBase.cs:527-559`

`TCPSend` 方法用 `UTF8.GetBytes(str)` 直接发送，无帧长度头、无 MBAP。

### 影响
- §A.1 / §B.1 / §C.1 注记 "TCP 6030 是 UTF8 JSON（换行分隔），不是 ModBus TCP"（已回填）
- §A.1 仍标注 "MBAP 处理待澄清"（Blocker B8）

### 遗留
仍需原开发者**最终确认**（见待澄清清单 Q-P07）：
- 是否真的没有 MBAP？
- 6020 的 ModBus 帧是直接跑 RTU 还是自定义了什么？
