# 江苏润盛 IoT SCADA 平台 — 系统架构说明

> 适合快速了解整个项目是什么、由哪些部分组成、Docker 在里面起什么作用。

---

## 1. 这是什么系统？

这是一个**工业设备远程监控平台（SCADA）**，用于石油/工业场景：采油机、保温电气、液位温湿度等设备的远程监控、告警和控制。

- 设备通过 **RS485 / DTU（4G）** 上传数据
- 工人/管理员通过**浏览器或微信**查看实时数据、接收告警、发送控制指令

## 3. 四个核心进程

| 进程 | 代码位置 | 语言/框架 | 职责 |
|------|----------|-----------|------|
| `ruisheng-gw` | `ruisheng-gw/` | Python asyncio + pymodbus | 接收设备 TCP 连接，解析 ModBus 协议，写数据到 Redis 和 PostgreSQL |
| `ruisheng-api` | `ruisheng-api/` | Python FastAPI + uvicorn | 提供 REST API 和 WebSocket，处理业务逻辑，向浏览器推送实时数据 |
| `ruisheng-web` | `ruisheng-web/` | Vue 3 + Vite | 前端页面，浏览器中运行，通过 Nginx 加载 |
| `Nginx` | Windows 安装 | Nginx | 反向代理 + TLS 终止，把不同路径分发到 API 或静态前端 |

**基础设施（数据层）：**

| 组件 | 职责 |
|------|------|
| **PostgreSQL 15 + TimescaleDB** | 持久化存储：用户、设备、历史时序数据（按月分表）、告警记录 |
| **Redis** | 实时总线：gw 把最新点值 Pub/Sub 推给 api，api 推给浏览器 WebSocket |

## 4. 数据流（三条主线）

### 4.1 实时采集流（gw → 数据库 → 浏览器）

```
设备 → DTU → TCP 连接 → ruisheng-gw
  → 解析 ModBus 帧
  → 写 Redis channel:realtime:{dev_number}   （实时缓存）
  → 写 PostgreSQL point_data_history         （历史记录）
  → ruisheng-api 订阅 Redis，推 WebSocket 给浏览器
```

### 4.2 控制流（浏览器 → 设备）

```
浏览器 → REST POST /api/control/command
  → ruisheng-api 写 Redis stream:control:cmd
  → ruisheng-gw 消费，发 ModBus 写命令给设备
  → 设备响应 → gw 写 control:result 回 api → 推 WebSocket 给浏览器
```

### 4.3 告警流

```
ruisheng-gw 判断阈值超限
  → 写 Redis stream:alarm:fired
  → ruisheng-api 消费，调用短信/语音/邮件/微信通知
  → 告警记录写入 PostgreSQL
  → WebSocket 推送告警到浏览器
```

## 5. Docker 在这里起什么作用？

### 5.1 结论先说

> **Docker 只用于本地开发环境**，不用于生产部署。
> 生产部署目标是 **Windows Server**，所有进程以 Windows 服务形式运行。

### 5.2 Docker 的角色（开发阶段）

```
开发者电脑
├── Python 源码（本地跑，不在容器里）
│     ├── ruisheng-gw   → uv run python -m ruisheng_gw
│     └── ruisheng-api  → uv run uvicorn ...
│
└── docker-compose.dev.yml  ← Docker 只管这两个基础设施
      ├── postgres 容器   (timescale/timescaledb:2.16.1-pg15)
      │     端口 5432，数据卷 ruisheng-pgdata
      └── redis 容器      (redis:7-alpine)
            端口 6379，数据卷 ruisheng-redisdata
```

**Docker 做的事：**
- 一条命令（`uv run task up`）把 PostgreSQL + Redis 启动好
- 不需要开发者在本机安装 PostgreSQL 或 Redis
- 数据持久化在 named volume，不会因为重启丢失

**Docker 不做的事：**
- 不运行 `ruisheng-gw`（开发时直接 `uv run python -m ruisheng_gw` 本地跑）
- 不运行 `ruisheng-api`（开发时直接 `uv run uvicorn` 本地跑）
- 不运行前端（开发时 `pnpm dev` 本地跑）
- 不用于生产

### 5.3 docker-compose.dev.yml 全文

```yaml
name: ruisheng-dev

services:
  postgres:
    image: timescale/timescaledb:2.16.1-pg15   # PostgreSQL + TimescaleDB 插件
    container_name: ruisheng-postgres-dev
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: ruisheng_dev
      POSTGRES_PASSWORD: ruisheng_dev
      POSTGRES_DB: ruisheng
    volumes:
      - ruisheng-pgdata:/var/lib/postgresql/data
      - ./seeds:/seeds:ro                        # 种子 SQL 可挂载进去手动执行

  redis:
    image: redis:7-alpine
    container_name: ruisheng-redis-dev
    ports:
      - "6379:6379"
    command: redis-server --appendonly yes --requirepass dev-redis-pw
    volumes:
      - ruisheng-redisdata:/data

volumes:
  ruisheng-pgdata:
  ruisheng-redisdata:
```

## 6. 代码仓库结构

```
ruisheng-scada/（Git 仓库）
│
├── ruisheng-shared/        # 共享包：ORM 模型、枚举、错误码、常量
├── ruisheng-gw/            # Plan 1：采集网关
│   └── src/ruisheng_gw/
│       ├── transport/      # TCP Server，处理设备 TCP 连接
│       ├── protocol/       # ModBus RTU 协议解析
│       ├── domain/         # 告警引擎、注册表、督导器
│       ├── persistence/    # 写 PostgreSQL（批量写入 + WAL 重放）
│       ├── pubsub/         # 写 Redis（发布实时点值、告警事件）
│       └── scheduler/      # 轮询调度器
│
├── ruisheng-api/           # Plan 2：Web API
│   └── src/ruisheng_api/
│       ├── api/            # FastAPI 路由（设备、告警、用户、控制……）
│       ├── services/       # 业务逻辑层
│       ├── db/             # SQLAlchemy ORM + 仓储层
│       └── pubsub/         # Redis 订阅、WebSocket 推送
│
├── ruisheng-web/           # Plan 3：Vue 3 前端
│   └── src/
│       ├── views/          # 页面（实时监控、历史、告警、组态……）
│       ├── stores/         # Pinia 状态管理
│       ├── ws/             # WebSocket 客户端（指数退避重连）
│       └── api/            # Axios HTTP 客户端
│
├── alembic/                # 数据库迁移脚本
├── seeds/                  # 开发环境种子数据
├── docker-compose.dev.yml  # 开发环境 Docker（只有 PG + Redis）
└── docs/                   # 设计文档、计划文档
```

## 7. 生产部署（Plan 4，待实施）

生产环境**不使用 Docker**，所有组件以 Windows 服务运行：

| 组件 | 安装方式 |
|------|----------|
| PostgreSQL 15 + TimescaleDB | Windows 原生安装，作为 Windows 服务 |
| Redis | Windows 原生安装，作为 Windows 服务 |
| ruisheng-gw | Python，用 `nssm` 注册为 Windows 服务 |
| ruisheng-api | Python uvicorn，用 `nssm` 注册为 Windows 服务 |
| ruisheng-web | `pnpm build` 打包后，由 Nginx 静态托管 |
| Nginx | Windows 版 Nginx，作为 Windows 服务 |
